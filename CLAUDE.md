# CLAUDE.md — Recession-Nowcast builder instructions

Project-specific conventions and gotchas. Read before changing the pipeline.

## The one rule: no look-ahead
**Never evaluate on `current.csv` (final-revised) data as if it were real-time.**
Every backtest decision at month `tau` must use only observations available by
`tau`. This is enforced by `pipeline/panel.py::VintageStore` and asserted in
`tests/test_no_lookahead.py`. Two backends:
- **vintage** — genuine FRED-MD dated vintages (`data/raw/fredmd/<vt>.csv`).
- **publication-lag** — when real vintages are unavailable, availability is
  approximated by a per-series lag (`GROUP_LAG` in `panel.py`). This removes
  *timing* look-ahead, not *revision* look-ahead; the vintage-vs-final comparison
  measures the residual.

## Data mode (important)
The pipeline stamps every artefact with `data_mode`:
- `live` — real series fetched from FRED / StatCan / BoC.
- `demo-synthetic` — a deterministic synthetic panel (real series IDs + real NBER
  dates + embedded signal) generated by `01_fetch/make_demo_panel.py` when the live
  endpoints are unreachable. The dashboard and report show a prominent banner.
  **Synthetic results are illustrative only — never quote them as empirical.**
Run `fetch_fred_series.py` where FRED is reachable to get `live` mode; no
downstream code changes.

## Series & transforms
- Panel series + FRED-MD `tcode` stationarity transforms live in `config.yaml`
  (`us.panel`). `1` level, `2` Dx, `3` D2x, `4` log, `5` Dlog, `6` D2log,
  `7` D(x_t/x_{t-1}-1). Applying tcodes is mandatory — never mix levels and growth
  rates into the factors (`panel.py::apply_tcode`).
- Labels: US = `USREC` (NBER). Canada = `data/manual/cdhowe_recessions.csv`
  (C.D. Howe chronology, one source URL per row).

## Models
- Benchmarks: term-spread probit (`bench_spread_probit.py`, statsmodels Probit),
  real-time Sahm (`bench_sahm.py`).
- Primary: dynamic factor model. The **full-sample** fit uses statsmodels
  `DynamicFactorMQ` (mixed-frequency Kalman/EM) in `dfm_statespace.py`. The
  **backtest** uses a fast PCA-EM factor approximation (`models.py::pca_em_factors`)
  so hundreds of expanding-window refits stay tractable; both are documented and
  the choice is stated in the report.
- Penalised MIDAS = elastic-net logistic on multi-lag indicators (`midas_logit.py`).
- Nonlinear comparator = XGBoost on factors (`ml_gbm.py`).
- In-loop estimators use sklearn logistic (numerically stable across refits); the
  standalone benchmark uses a genuine probit for the coefficient table.

## Evaluation
- Pseudo-real-time expanding window in `backtest.py`. Metrics: Brier, log-loss,
  AUC, PR-AUC, ECE + reliability curve. CIs via stationary block bootstrap
  (`metrics.py`). Predictive-ability via Diebold-Mariano and Giacomini-White.
- **COVID** (`2020-02..2020-08`) is reported both included and excluded; never let
  it drive the headline (`metrics_ex_covid`).
- Three configs: US real-time (headline), US final (skill upper bound), Canada
  real-time.

## Uncertainty (mandatory)
`uncertainty.py` produces a **probability band**, not a point: parameter
uncertainty (block-bootstrap refit) × factor-state uncertainty (MC over smoothed
factor std). The dashboard shows the shaded band and a current-month gauge.

## Determinism & artefacts
- `SEED = config.yaml:seed` (20260615), recorded in results JSON.
- `data/raw/`, `data/interim/` are gitignored; `data/processed/`, `data/facts/`,
  `data/manual/` and `site/data/` are committed (small) so `--offline` rebuilds.
- All `site/data/*.json` must pass the browser-parseable guard (no NaN/Infinity);
  `common.write_json` enforces it and the CI re-checks.

## Performance notes
- The backtest refits every model every month across 3 configs; the elastic-net
  (saga) refit dominates. For faster local iteration raise `eval.start`, increase
  an internal `refit_every`, or lower `bootstrap.n_boot`. The committed run uses
  the config values verbatim.
